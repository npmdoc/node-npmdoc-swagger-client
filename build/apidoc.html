HTTP Client
-----------

`Swagger.http(req)` exposes a [Fetch-like interface](https://github.com/matthew-andre\
ws/isomorphic-fetch) with a tweak: allowing `url` in the request object so that it can be passed around and mutated. It extends \
Fetch to support request and response interceptor and perform response & headers serialization. This method could be overri\
dden to change how SwaggerJS performs HTTP requests.

Swagger Spec Resolver
---------------------

`Swagger.resolve({url, spec, http})` resolves `$ref`s (JSON-Refs) with the objects th\
ey point to.

```js

Swagger.resolve({url, spec, http}).then((resolved) => {
  resolved.errors // resolution errors, if any
  resolved.spec   // the resolved spec
